# Debug Changelog — Algorithme de matching SwipeScreen

## Problème initial
L'algorithme proposait des offres hors secteur (ex: vente au lieu d'informatique) car il récupérait TOUTES les offres et les triait par score sans jamais filtrer les non-pertinentes.

## Cause racine
Dans `_loadData()`, la requête Supabase récupérait tous les jobs (`from('jobs').select()`) puis les triait par `_calculateMatchScore`. Les offres avec score -100 (aucun match) restaient dans la liste et étaient montrées après les bons matchs.

---

## Session 1 — Fix incorrect (reverted)
Filtrage server-side via `.contains('tags', [skill])` — échoué car `tags` est `TEXT[]` et non JSONB.

## Session 2 — Fix client-side filtrage (appliqué)
Filtrage `score > 0` après calcul des scores. Ne change pas la requête Supabase.
